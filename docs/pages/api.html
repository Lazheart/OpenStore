API & Gateway

Gateway, microservicios y contratos HTTP

Todas las peticiones del frontend pasan por store-service. Ese servicio enruta a user-service, shop-service y product-service, valida ownership y desacopla al cliente de la topología interna.

store-service orquesta toda la capa de API

Una sola URL pública, cuatro servicios internos

El frontend no habla con los microservicios de forma directa. store-service actúa como fachada pública: recibe las solicitudes, redirige al servicio correcto y coordina cadenas de llamadas cuando una operación toca varias entidades.

Responsabilidad de cada microservicio

store-service

Gateway público. Resuelve login, registro, crea/elimina tiendas, proxea el catálogo y expone endpoints de analytics consultando Athena.

user-service

Gestiona autenticación, registro, perfiles, actualización de usuario y control de suscripciones. Fuente de verdad para roles y datos del usuario.

shop-service

Administra tiendas, membresías y temas del storefront. Aplica límites por plan y valida que el owner sea el correcto antes de modificar.

product-service

Maneja el catálogo de productos, las imágenes subidas a S3 y las operaciones CRUD ligadas a una tienda específica.

Endpoints por servicio

store-service (gateway público)

  • Auth: POST /auth/login, POST /auth/register, GET /me, PATCH /me, POST /verify
  • Suscripción: PATCH /user/me/subscription, PATCH /user/{user_id}/subscription
  • Tiendas: POST /openshop/shop, GET /shops, GET /shops/{id}, PATCH /shop/id/{id}, DELETE /shop/id/{id}, GET|PUT /shop/{id}/theme
  • Productos: GET|POST /shops/{id}/products, PATCH|DELETE /shops/{id}/products/{pid}
  • Analytics: GET /analytics/productos-por-tienda, /analytics/usuarios-por-tienda, /analytics/membresias-por-tienda, /analytics/resumen-tiendas

user-service (interno)

  • POST /auth/register, POST /auth/{shopId}/register
  • POST /auth/login, POST /auth/{shopId}/login
  • GET /me, PATCH /me, POST /verify
  • GET /users, GET /users/{id}, PUT /users/{id}, DELETE /users/{id}
  • PATCH /user/{id}/subscription, PATCH /user/me/subscription

shop-service (interno)

  • GET /health, GET /shops, GET /shop/:shopId
  • POST /openshop/shop
  • GET /shop/name/:name, GET /shop/owner/:ownerId
  • PATCH /shop/:shopId, PUT|GET /shop/:shopId/theme
  • DELETE /shop/id/:shopId, POST /shops/:id/memberships

product-service (interno)

  • GET /healthcheck
  • GET /shops/{shop_id}/products
  • POST /shops/{shop_id}/products
  • PATCH /shops/{shop_id}/products/{product_id}
  • DELETE /shops/{shop_id}/products/{product_id}
