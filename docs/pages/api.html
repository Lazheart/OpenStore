OpenStore API

Gateway, microservicios y contratos HTTP

Todas las request del frontend pasan por store-service. Ese servicio mapea las operaciones a user-service, shop-service y product-service, valida ownership, expone analytics y desacopla al cliente de la topología interna.

Punto de entrada

store-service orquesta toda la capa de API

El frontend no habla con los microservicios internos de forma directa. Store-service actúa como fachada pública: recibe las solicitudes, redirige al servicio correcto y coordina cadenas de llamadas cuando una operación toca varias entidades.

Idea clave: una sola URL pública simplifica la integración, reduce el desacoplamiento y deja la lógica de composición en el backend.

Responsabilidad

Qué hace cada microservicio

store-service

Es el gateway público. Resuelve login y registro, crea y elimina tiendas, proxyea consultas al catálogo, publica endpoints de analytics y expone la documentación mapeada de todos los servicios.

user-service

Gestiona autenticación, registro, perfiles, actualización de usuario y control de suscripciones. Es la fuente de verdad para roles y datos del usuario.

shop-service

Administra tiendas, membresías y temas del storefront. Aplica límites por plan, valida ownership y persiste la relación tienda-suscriptor.

product-service

Maneja el catálogo de productos, las imágenes subidas a S3 y las operaciones CRUD ligadas a una tienda.

Bases de datos

Qué tipo de base usa cada servicio

Servicio Base de datos Motivo
user-service PostgreSQL Relaciones de usuario, roles, perfiles y suscripciones.
shop-service MySQL Tiendas, membresías y configuración del storefront.
product-service MongoDB Catálogo flexible con documentos de productos y metadatos.
store-service No aplica Solo orquesta y consulta servicios externos/Athena.

Resumen

Si quieres integrar otra aplicación con OpenStore, usa store-service como punto de entrada. Solo los servicios internos conocen sus detalles de persistencia y ningún cliente necesita hablar directo con PostgreSQL, MySQL o MongoDB.
