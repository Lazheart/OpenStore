OpenStore Data ingest

Cómo se construyó la ingesta y por qué usamos boto3

La ingesta prepara un conjunto de datos reproducible para analítica. Los scripts extraen información desde PostgreSQL, MySQL y MongoDB, la transforman a formatos portables y la publican en S3 para que Athena pueda consultarla después.

Pipeline

La ingesta toma datos ya normalizados y los lleva a S3

Primero se generan datos sintéticos para poblar las bases. Después, los jobs de ingestión extraen cada fuente por separado, serializan el resultado y lo suben a un bucket S3 en un formato que Athena pueda leer.

Cómo se hizo

Flujo de ingesta paso a paso

1. Semilla de datos

Los scripts seed.py y seed_direct.py usan Faker para generar usuarios, tiendas, membresías y productos con datos coherentes y reproducibles.

2. Extracción por dominio

Cada script de la carpeta ingest-* extrae una sola fuente: usuarios desde PostgreSQL, tiendas y membresías desde MySQL, y productos desde MongoDB.

3. Serialización

Los datos se convierten a CSV o JSON Lines según el tipo de fuente. JSON Lines es útil para consultas analíticas y para mantener un registro por línea.

4. Publicación en S3

Los archivos se suben con boto3 a rutas estables como users/users.csv, shops/shops.csv, shops/memberships.csv y products/products.json.

5. Consumo analítico

store-service consulta Athena para construir métricas por tienda, usuarios y membresías sin depender de consultas ad hoc a cada base operativa.

Por qué boto3

Qué resuelve boto3 en este proyecto

Integración nativa con AWS

Permite hablar con S3 y Athena usando el SDK oficial de Python, sin depender de scripts manuales o procesos externos adicionales.

Autenticación por IAM

Las credenciales se resuelven con la identidad de ejecución. Eso hace más simple mover el mismo código entre local, EC2 o contenedores.

Subida simple de archivos

La API de upload_file facilita mandar CSV o JSON al bucket sin tener que reconstruir todo el proceso de carga a mano.

Encaje con el resto del stack

Python ya es la base de la ingesta y de varios servicios del proyecto, así que boto3 encaja sin introducir otra dependencia de plataforma.
