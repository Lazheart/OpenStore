Data Ingest

Pipeline de ingesta con boto3, S3 y Athena

Scripts Python que extraen datos de las tres bases operativas, los transforman a formatos portables y los publican en S3 para que store-service los consulte a través de Athena.

Flujo de ingesta paso a paso

1. Semilla de datos

seed.py y seed_direct.py usan Faker para generar usuarios, tiendas, membresías y productos con datos coherentes y reproducibles, llenando las bases de cada servicio.

2. Extracción por dominio

Cada script en ingest-*/ extrae una sola fuente: usuarios desde PostgreSQL, tiendas y membresías desde MySQL, productos desde MongoDB. Un script, una responsabilidad.

3. Serialización

Los datos se convierten a CSV (usuarios, tiendas, membresías) o JSON Lines (productos). JSON Lines mantiene un registro por línea, ideal para consultas analíticas.

4. Publicación en S3

Los archivos se suben con boto3.upload_file a rutas estables: users/users.csv, shops/shops.csv, shops/memberships.csv y products/products.json.

5. Consumo analítico vía Athena

store-service consulta Athena para construir métricas por tienda, usuarios y membresías sin depender de consultas ad hoc a las bases operativas.

Por qué boto3

SDK oficial de Python para AWS

Habla con S3 y Athena sin depender de scripts manuales o procesos externos. El mismo código funciona en local, EC2 o contenedores.

Autenticación por IAM

Las credenciales se resuelven con la identidad de ejecución: variables de entorno en local, roles de instancia en EC2, simplificando el movimiento entre entornos.

Subida simple de archivos

upload_file envía CSV o JSON al bucket sin reconstruir el proceso de carga a mano ni gestionar multipart manualmente.

Encaje con el stack

Python ya es la base de store-service y product-service, así que boto3 no introduce otra dependencia de plataforma.
