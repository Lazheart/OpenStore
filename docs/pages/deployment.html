OpenStore Deployment

CloudFormation, Docker y despliegue reproducible

El proyecto usa CloudFormation para que la infraestructura se pueda recrear de forma determinista. Los servicios se empaquetan con Docker, las bases de datos se levantan con Docker Compose y el frontend se publica por Amplify.

Por qué CloudFormation

La infraestructura debe poder recrearse igual en cualquier cuenta

CloudFormation deja la infraestructura declarada en un template versionable. Eso evita cambios manuales difíciles de rastrear, permite repetir la misma topología en otra cuenta y hace más simple revisar qué recursos necesita realmente OpenStore.

Buenas prácticas

Si trabajas con cuentas de estudiantes

Usa un rol dedicado

Es mejor crear un role específico para ejecutar la plantilla que usar la cuenta raíz. Así reduces el riesgo y delimitas qué acciones pueden realizar las pruebas.

Aplica least privilege

Limita permisos a EC2, ELB, S3, CloudFormation y Amplify solo a lo necesario para el despliegue. Menos permisos equivalen a menos superficie de error.

Microservicios

Cada servicio del backend tiene su propio Dockerfile, lo que permite construir imágenes independientes y desplegarlas de forma aislada.

Local dev

El stack local usa Compose para levantar las bases y después los servicios, conservando la misma separación que existe en AWS.

Orden recomendado: primero bases de datos, después APIs y al final el frontend.

Docker Compose

Cómo se levantan bases y APIs en tu máquina

Bases de datos

database/docker-compose.yml levanta PostgreSQL en 5432, MySQL en 3307 y MongoDB en 27017. Cada contenedor conserva su volumen para no perder datos entre reinicios.

APIs

backend/docker-compose.yml arranca user-service, shop-service, product-service y store-service con healthchecks y variables de entorno predefinidas.

Archivo Qué arranca Nota
database/docker-compose.yml PostgreSQL, MySQL, MongoDB Una base por servicio persistida en volúmenes locales
backend/docker-compose.yml user, shop, product y store services Conexiones internas y healthchecks listos

Frontend

Despliegue del frontend

Build

La carpeta frontend se compila con Vite usando pnpm install y pnpm run build. El resultado sale en dist.

Amplify

CloudFormation crea la app de Amplify y apunta al repo del proyecto para que el frontend se despliegue automáticamente desde la rama configurada.

Objetivo: el frontend debe poder publicarse sin tocar manualmente el servidor cada vez que cambia una vista.
