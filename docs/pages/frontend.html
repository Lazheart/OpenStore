Frontend

React + Vite + TailwindCSS 4

La interfaz está construida con React 19 y servida por Vite. La navegación se resuelve con React Router; el estado de autenticación vive en AuthContext y el tema visual en ThemeConfig.

Stack tecnológico

React 19 Vite TailwindCSS 4 TypeScript React Router

El frontend no es una vista monolítica: cada pantalla está separada por ruta y componente, permitiendo que la app crezca sin hacer inmanejable el árbol de UI.

Tipos de rutas y renderizado

Rutas públicas

HomePage vive dentro de PublicLayout. Esa capa controla el header público, el cambio de tema y la navegación base de la landing.

Rutas protegidas

ProtectedRoute valida si existe sesión activa. Si no hay usuario autenticado, redirige a /login antes de permitir entrar a owner, admin o perfil.

Panel interno

AppLayout agrupa las páginas que usan sidebar y topbar: /owner, /admin y /profile.

Storefront público

La ruta dinámica /:shopSlug?id=… carga la tienda, su tema visual y su catálogo para renderizar el template público adecuado.

Autenticación por rol

Owner y Admin

El formulario de login envía email y contraseña al backend. La respuesta se guarda en AuthContext y en localStorage, luego redirige según el rol: OWNER al panel del dueño, ADMIN al panel de administración.

Cliente de tienda

El gateway soporta rutas con shopId para registrar o autenticar usuarios de tienda. Eso permite cubrir escenarios de cliente sin romper el login central.

Persistencia de sesión: el token queda en localStorage y la app vuelve a hidratar el usuario al recargar la página.

Componentes clave

Layouts

PublicLayout, AppLayout y el layout del storefront separan cada contexto visual de la aplicación.

AuthContext

Centraliza usuario, funciones de login y logout para que cualquier pantalla pueda leer o modificar la sesión activa.

ThemeConfig

Mantiene el tema visual activo de cada storefront y permite alternarlo en tiempo real desde la interfaz pública.
