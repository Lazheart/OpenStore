OpenStore Documentation

Plataforma de tiendas virtuales basada en microservicios. Centraliza autenticación, catálogo de productos y gestión de tiendas con una arquitectura escalable y despliegue reproducible.

¿Qué es OpenStore?

Crea y administra tiendas virtuales desde un solo sistema

OpenStore permite a los dueños (owners) crear y configurar tiendas en línea con catálogo de productos, temas visuales personalizables y control de usuarios por tienda. Los clientes se registran e inician sesión dentro del contexto de cada tienda, manteniendo el aislamiento de datos entre negocios.

React 19 Vite TailwindCSS 4 Spring Boot FastAPI Express + Prisma Docker AWS

¿Cómo funciona? Arquitectura de microservicios

El sistema está dividido en servicios independientes que se comunican entre sí a través de store-service, el gateway público. Cada servicio es responsable de un único dominio.

🔀store-service — Gateway

Punto de entrada público. Recibe todas las peticiones del frontend, las enruta al servicio correcto y orquesta operaciones que involucran múltiples dominios. Corre en el puerto 8004.

👤user-service — Autenticación

Gestiona registro, login, perfiles, roles y suscripciones. Construido con Spring Boot y PostgreSQL. Es la fuente de verdad para identidad y permisos. Puerto 8080.

🏪shop-service — Tiendas

Administra tiendas, membresías y temas visuales del storefront. Construido con Express + Prisma sobre MySQL. Aplica límites por plan de suscripción. Puerto 3000.

📦product-service — Catálogo

Maneja el catálogo de productos, subida de imágenes a S3 y operaciones CRUD por tienda. Construido con FastAPI y MongoDB. Puerto 8000.

Principio clave: el frontend nunca habla directamente con user-service, shop-service o product-service. Toda petición pasa por store-service, que valida autenticación y ownership antes de delegar.

Estructura del proyecto

OpenStore/
├── frontend/            # React 19 + Vite + TailwindCSS 4 + TypeScript
│   ├── src/
│   │   ├── components/    # Componentes reutilizables
│   │   ├── pages/         # HomePage, LoginPage, OwnerPanel, AdminPanel…
│   │   ├── context/       # AuthContext, ThemeConfig
│   │   └── App.tsx        # Router principal (React Router)
│   └── package.json
├── backend/
│   ├── store-service/      # Gateway FastAPI — puerto 8004
│   ├── user-service/       # Auth Spring Boot — puerto 8080
│   ├── shop-service/       # Tiendas Express/Prisma — puerto 3000
│   ├── product-service/    # Catálogo FastAPI — puerto 8000
│   └── docker-compose.yml # Orquestación de servicios
├── database/
│   └── docker-compose.yml  # PostgreSQL · MySQL · MongoDB
├── ingest/               # Scripts Python + boto3 para S3 y Athena
├── cloudformation/       # Infraestructura AWS como código
├── docs/                 # Esta documentación
└── .github/workflows/   # CI: build y push de imágenes Docker
